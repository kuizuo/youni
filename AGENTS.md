# AGENTS.md

Guidance for coding agents working in this repository.

## Project Snapshot

Youni is a Bun-powered TypeScript monorepo generated from Better-T-Stack and extended into a social content product.

- `apps/web`: React 19 + Vite + TanStack Router admin console. The public home route redirects to `/admin`.
- `apps/native`: Expo Router + React Native mobile app for the user-facing feed, publishing, search, profile, messages, and note detail flows.
- `apps/server`: Hono server exposing Better Auth routes, oRPC routes, OpenAPI reference routes, and an `/ai` streaming endpoint.
- `packages/api`: Shared oRPC router and business logic. Add or change API procedures here.
- `packages/db`: Drizzle Cloudflare D1 schema, migrations, seed script, and database client.
- `packages/auth`: Better Auth configuration shared by server and clients.
- `packages/env`: Environment variable loading and validation for server, web, native, and local Cloudflare-style runtime.
- `packages/ui`: Shared HeroUI styles and global CSS exports.
- `packages/infra`: Cloudflare deployment through Alchemy.
- `packages/config`: Shared TypeScript config.

## Tooling

- Use `bun`, not npm, yarn, or pnpm.
- Package manager version is `bun@1.3.10`.
- Root scripts are routed through Turborepo.
- Formatting and lint fixes use Biome, configured with tab indentation and double quotes.
- TypeScript is strict. The shared config enables unused checks and `noUncheckedIndexedAccess`.

## Common Commands

Run from the repository root unless noted.

- Install dependencies: `bun install`
- Start everything in development: `bun run dev`
- Start web only: `bun run dev:web`
- Start Cloudflare web/server stack: `bun run dev:server`
- Start native only: `bun run dev:native`
- Build all apps/packages: `bun run build`
- Type-check all configured packages: `bun run check-types`
- Format and lint with safe fixes: `bun run check`
- Push database schema to D1 directly: `bun run db:push`
- Generate database migration files: `bun run db:generate`
- Run database migrations: `bun run db:migrate`
- Seed database: `bun run db:seed`
- Open Drizzle Studio: `bun run db:studio`
- Deploy Cloudflare resources: `bun run deploy`
- Destroy Cloudflare resources: `bun run destroy`

## Local Runtime

- Server defaults to `http://localhost:3000`.
- Web dev server uses `http://localhost:3001`.
- Expo commonly uses `http://localhost:8081`.
- Before starting a dev server, check whether the expected port is already listening. If the matching service is already running, reuse it instead of starting another process.
- Web expects `VITE_SERVER_URL`.
- Native expects `EXPO_PUBLIC_SERVER_URL`.
- Server expects the Cloudflare D1 `DB` binding from Alchemy plus values such as `CORS_ORIGIN`, `BETTER_AUTH_SECRET`, `BETTER_AUTH_URL`, Google auth/model keys, and `ADMIN_EMAILS`.
- Env files currently exist under `apps/server/.env`, `apps/web/.env`, `apps/native/.env`, and `packages/infra/.env`. Do not print secrets in logs or responses.

## Architecture Notes

- API clients in web and native both consume `AppRouterClient` from `@youni/api/routers/index`.
- Web RPC calls live in `apps/web/src/utils/orpc.ts` and use browser credentials.
- Native RPC calls live in `apps/native/utils/orpc.ts` and manually forward Better Auth cookies outside web.
- Auth is configured in `packages/auth/src/index.ts`; do not create separate auth setup in app folders.
- Database access should go through `createDb()` from `@youni/db`.
- Shared database tables are exported from `packages/db/src/schema/index.ts`.
- Admin-only API behavior belongs in `packages/api/src/routers/admin.ts`; admin access is based on `ADMIN_EMAILS`.
- User-facing social behavior belongs in `packages/api/src/routers/social.ts`.
- Notification behavior belongs in `packages/api/src/routers/notifications.ts` and `packages/api/src/lib/notifications.ts`.
- The `todo` router and schema still exist as starter-stack functionality; do not expand them unless the task explicitly concerns todo behavior.

## Web App Conventions

- Routes live in `apps/web/src/routes`.
- `apps/web/src/routeTree.gen.ts` is generated by TanStack Router. Do not edit it by hand.
- Admin layout and reusable admin page framing live in `apps/web/src/components/admin-shell.tsx`.
- Prefer existing HeroUI and HeroUI Pro components before adding custom primitives.
- Keep admin screens dense, clear, and operational rather than marketing-like.
- Import app-local web modules through `@/`.

## Native App Conventions

- Routes live in `apps/native/app`.
- Main tabs are under `apps/native/app/(tabs)` and use `FloatingTabBar`.
- Feature screens live under `apps/native/components`.
- Use HeroUI Native / HeroUI Native Pro components where they already fit.
- Use `fireHaptic()` for intentional press feedback in interactive flows where the existing code does so.
- Import app-local native modules through `@/`.
- Expo config is split between `app.json` and `app.config.js`; Google Sign-In plugin setup is added dynamically in `app.config.js`.

## API and Data Changes

- Define input validation with Zod at the router boundary.
- Use `protectedProcedure` for endpoints that require a logged-in user.
- Keep database writes in API routers or small helpers under `packages/api/src/lib`.
- When changing database schema, update Drizzle schema first, then generate or update migrations with the repo's DB commands.
- Do not hand-edit existing migration snapshots unless a migration task specifically requires it and the generated output has been reviewed.
- Keep D1/SQLite-specific behavior in `packages/db`.

## Styling and UI

- Web global styles come from `packages/ui/src/styles/globals.css` through `@youni/ui`.
- Native global styles are in `apps/native/global.css`.
- Biome sorts class names for `clsx`, `cva`, and `cn`; expect `bun run check` to rewrite class order.
- Preserve existing Chinese UI copy style in user-facing screens unless a task asks for a language change.

## Verification Expectations

Before reporting completion, run the narrowest meaningful checks for the change:

- General TypeScript or shared logic changes: `bun run check-types`
- Formatting/linting or broad edits: `bun run check`
- Web UI changes: verify whether `http://localhost:3001` is already running; reuse it if available, otherwise run `bun run dev:web`, then open the affected route and verify rendering and interaction.
- Server/API changes: verify whether `http://localhost:3000` is already running; reuse it if available, otherwise run `bun run dev:server`, then exercise the affected route/procedure where practical.
- Native UI changes: verify whether the Expo server is already running, commonly on `http://localhost:8081`; reuse it if available, otherwise run `bun run dev:native` or the relevant Expo target, then verify the changed screen.
- Database schema changes: run the relevant Drizzle command and inspect generated migration output.

If a full check cannot run because required services or secrets are unavailable, state that plainly and run the best local substitute.

## Safety Rules

- Do not overwrite user changes. Check `git status --short` before and after meaningful edits.
- Do not commit, push, deploy, destroy infrastructure, or run destructive database operations unless the user asks.
- Do not expose `.env` values, tokens, cookies, or generated secrets.
- Keep changes scoped to the requested behavior.
- Prefer the repository's existing patterns over introducing new frameworks or parallel abstractions.
