# ADR: Skia-first 发布页图片编辑与导出策略

## 状态

已采用

## 背景

发布页需要支持对已选图片做裁剪、变换、文字贴纸、画笔和模糊遮挡处理，体验接近 X/Twitter 发帖时的图片编辑流程。

现有发布、草稿和上传接口只接收图片 URL 列表，不保存编辑图层、裁剪参数或其他编辑历史。

## 决策

图片编辑器的预览、手势编辑、画布渲染和保存导出以 `@shopify/react-native-skia` 为核心实现。

进入编辑器时先用 `expo-image-manipulator` 把输入图片转成本地 JPEG，再交给 Skia 读取和编辑。`expo-image-manipulator` 不参与裁剪、旋转、绘制、模糊遮挡或最终效果生成；它只负责把相册里的 HEIC、兼容模式图片或其他系统 URI 变成 Skia 更稳定可读的本地 JPEG。

用户点击完成时，把当前 Skia 画布导出为新的本地 JPEG，并把该本地文件作为普通待上传图片交给现有发布流程。

用户在裁切工具里点击确认时，如果裁切范围发生变化，编辑器会先用 Skia 按当前裁切区域生成一张新的内存图片，并把它作为后续文字、画笔、模糊和再次裁切的基础。这个步骤不写文件、不上传，只是让裁切结果在编辑器里立即可见。

导出时优先用 Skia 离屏画布按当前裁剪区域重新渲染，长边最多 2048px。若离屏渲染在个别环境失败，则退回当前画布截图导出，保证用户不会卡在无法完成编辑的状态。

不保存可二次编辑的图层历史。文字贴纸、画笔、模糊遮挡、裁剪和变换都会直接写入导出图。

## 影响

- 新发布图片和编辑后的草稿图片复用现有上传接口，不新增后端字段。
- 远程草稿图未被编辑时继续沿用原 URL，不重复上传。
- 远程草稿图被编辑后变成本地新图，下次保存或发布时重新上传。
- 取消编辑不会修改原图。
- 已确认的裁切会立即进入编辑画面，后续工具基于裁切后的结果继续编辑。
- GIF 暂不进入编辑器，仍可原样发布。
- 无法转成本地 JPEG，且 Skia 也无法直接读取的图片会提示用户换一张图片。
- 后续如果要支持二次编辑，需要新增图层历史数据结构和后端存储，不应复用当前导出图作为完整编辑状态。

## 备选方案

保存图层历史和编辑参数：可二次编辑，但需要新增后端字段、版本兼容和迁移策略，超出首版范围。

使用 `expo-image-manipulator` 作为主要编辑引擎：实现轻，但无法支撑贴纸、画笔、模糊遮挡和更顺滑的画布交互。

只使用系统图片编辑器：实现更轻，但无法覆盖文字贴纸、画笔和模糊遮挡等产品需要的编辑能力。
